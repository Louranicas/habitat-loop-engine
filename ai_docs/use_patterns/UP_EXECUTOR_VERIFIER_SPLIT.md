# UP_EXECUTOR_VERIFIER_SPLIT

Status: scaffold use-pattern contract. Runtime executor implementation is explicitly deferred until `begin M0` authorization.

Predicate ID: `HLE-UP-001`

## Intent

The Loop Engine must keep action and authority separate. An executor may propose, stage, or run a bounded workflow step only after the future M0 gate exists; it must not be able to certify its own success. A verifier consumes artifacts, receipts, hashes, and negative-control evidence from an independent surface and emits a PASS/BLOCKED verdict.

## Scaffold-time encoding

- `crates/substrate-types` owns neutral data types shared by both sides.
- `crates/substrate-emit` may define emission contracts and sample records, but it cannot depend on verifier internals.
- `crates/substrate-verify` owns verification contracts and negative-control expectations, but it cannot execute workflow actions.
- `schemas/receipt.schema.json` and `.deployment-work/receipts/` record claim authority without turning claims into proof.
- `HARNESS_CONTRACT.md` names the verifier boundary that future M0 work must honor.

## Required evidence before future PASS

1. A claim receipt names the actor that produced the artifact.
2. A separate verifier receipt names the verifier command, input hashes, output hash, and verdict.
3. The verifier receipt cannot be generated by the same binary or runtime role that produced the claim.
4. Negative controls demonstrate at least one false-pass fixture remains rejected.
5. The receipt graph links claim -> verifier -> superseding receipt if any correction occurs.

## Forbidden pattern

A single command that both performs the workflow action and writes FINAL_DEPLOYMENT_RECEIPT is invalid. Future implementations must fail closed if executor and verifier identity collapse.

## Review checklist

- Does the new surface preserve two-binary executor/verifier separation?
- Is every PASS backed by independent verifier evidence rather than prose confidence?
- Are negative controls present for the claim class?
- Does the receipt name source hashes and authority class?
