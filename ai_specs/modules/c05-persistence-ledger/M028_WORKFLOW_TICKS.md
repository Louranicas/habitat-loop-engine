# M028 Workflow Ticks — Tick Ledger for Freshness and Causality

> **Module ID:** M028 | **Cluster:** C05_PERSISTENCE_LEDGER | **Layer:** L02
> **Source:** `crates/hle-storage/src/workflow_ticks.rs`
> **Error Codes:** 2430–2431
> **Role:** Per-step tick ledger. Every significant event within a workflow run is recorded
> as a monotonically increasing tick. Ticks carry a `parent_tick_id` causality pointer,
> enabling verifiers to reconstruct the causal chain of events for a run. This satisfies
> framework §17.5's freshness model.

## `TickKind` Enum

```rust
/// Classifies the event that produced a tick.
/// Stored as a text column. Extend by adding variants — do not rename existing ones.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum TickKind {
    /// Run was initiated (`hle run` called).
    RunStart,
    /// A workflow step began execution.
    StepStart,
    /// A workflow step completed (PASS or FAIL — verdict in evidence_store).
    StepComplete,
    /// Execution paused awaiting human input.
    AwaitingHuman,
    /// Human provided input; execution resumed.
    HumanResolved,
    /// An evidence artifact was recorded (links to evidence_store).
    EvidenceRecorded,
    /// A verifier verdict was recorded (links to verifier_results_store).
    VerifierVerdictRecorded,
    /// Run reached a terminal status.
    RunComplete,
}

impl TickKind {
    #[must_use]
    pub const fn as_str(self) -> &'static str;

    /// # Errors
    /// Returns [`StorageError::TickInsert`] (2430) for unknown kind strings.
    pub fn from_str(s: &str) -> Result<Self, StorageError>;
}

impl std::fmt::Display for TickKind { /* delegates to as_str */ }
```

/// Deserialized row from `workflow_ticks`.

pub struct Tick {

    pub run_id: i64,

    pub tick_id: i64,

    /// Step identifier, if this tick is associated with a specific step.

    /// Unix epoch seconds — supplied by caller, not generated by the store.
    pub created_unix: i64,
    /// Causality pointer. `None` only for the first tick of a run (RunStart).
    /// All subsequent ticks must reference a prior tick in the same run.
    pub parent_tick_id: Option<i64>,
}

## `WorkflowTicksStore` Struct and Methods

```rust
#[derive(Debug, Clone)]
pub struct WorkflowTicksStore {
    pool: std::sync::Arc<dyn Pool>,
}

impl WorkflowTicksStore {
    #[must_use]
    pub fn new(pool: std::sync::Arc<dyn Pool>) -> Self;

    /// Insert a new tick row. Returns the assigned `tick_id`.
    ///
    /// Validates that `parent_tick_id`, if supplied, references an existing tick row
    /// within the same `run_id`.
    ///
    /// # Errors
    /// - [`StorageError::TickParentMissing`] (2431) if `parent_tick_id` is not found.
    /// - [`StorageError::TickInsert`] (2430) on SQL failure.
    pub fn insert(&self, tick: &NewTick) -> Result<i64, StorageError>;

    /// Fetch all ticks for a run, ordered by `tick_id` ascending.
    ///
    /// # Errors
    /// Returns [`StorageError::Storage`] (2499) on query failure.
    pub fn list_for_run(&self, run_id: i64) -> Result<Vec<Tick>, StorageError>;

    /// Fetch the most recent tick for a run.
    ///
    /// # Errors
    /// Returns `Ok(None)` if the run has no ticks yet.
    pub fn latest_for_run(&self, run_id: i64) -> Result<Option<Tick>, StorageError>;

    /// Reconstruct the causal chain from a given `tick_id` back to the RunStart tick.
    /// Returns the chain in chronological order (root first).
    ///
    /// # Errors
    /// - [`StorageError::TickParentMissing`] (2431) if a chain link is broken.
    /// - [`StorageError::Storage`] (2499) on query failure.
    pub fn causal_chain(&self, tick_id: i64) -> Result<Vec<Tick>, StorageError>;

    /// Count ticks of a given kind for a run.
    ///
    /// # Errors
    /// Returns [`StorageError::Storage`] (2499) on query failure.
    pub fn count_by_kind(&self, run_id: i64, kind: TickKind) -> Result<i64, StorageError>;

    /// Delete ticks for runs older than the retention window.
    /// Predicate: `created_unix < (strftime('%s','now') - retention_secs)`.
    ///
    /// # Errors
    /// Returns [`StorageError::Storage`] (2499) on query failure.
    pub fn delete_older_than(&self, retention_secs: i64) -> Result<usize, StorageError>;
}
```

## SQL Patterns

```sql
-- workflow_ticks table (defined in migrations/0002_ticks.sql — planned)
CREATE TABLE IF NOT EXISTS workflow_ticks (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    run_id         INTEGER NOT NULL REFERENCES workflow_runs(id),
    tick_id        INTEGER NOT NULL,  -- monotonic within run, assigned by AUTOINCREMENT
    kind           TEXT    NOT NULL,
    step_id        TEXT,
    created_unix   INTEGER NOT NULL,
    parent_tick_id INTEGER REFERENCES workflow_ticks(id)
) STRICT;

CREATE INDEX IF NOT EXISTS idx_ticks_run_id ON workflow_ticks(run_id);
CREATE INDEX IF NOT EXISTS idx_ticks_parent ON workflow_ticks(parent_tick_id);

-- insert
INSERT INTO workflow_ticks (run_id, tick_id, kind, step_id, created_unix, parent_tick_id)
VALUES (?1, (SELECT COALESCE(MAX(tick_id),0)+1 FROM workflow_ticks WHERE run_id=?1),
        ?2, ?3, ?4, ?5);

-- parent validation (before insert)
SELECT COUNT(*) FROM workflow_ticks WHERE id = ?1 AND run_id = ?2;

-- causal chain (recursive CTE)
WITH RECURSIVE chain(id, run_id, tick_id, kind, step_id, created_unix, parent_tick_id) AS (
    SELECT * FROM workflow_ticks WHERE id = ?1
    UNION ALL
    SELECT t.* FROM workflow_ticks t
    INNER JOIN chain c ON t.id = c.parent_tick_id
)
SELECT * FROM chain ORDER BY tick_id ASC;

-- TTL delete — retention_secs is a bound parameter
DELETE FROM workflow_ticks WHERE created_unix < (strftime('%s','now') - ?1);
```

## Causality Invariants

The tick table enforces these invariants, validated by the store on insert:

| Rule | Detail |
|------|--------|
| `RunStart` has no parent | `parent_tick_id` must be `NULL` |
| All other kinds require a parent | `parent_tick_id` must be `NOT NULL` |
| Parent must belong to same run | Validated by the FK + SELECT check before insert |
| `tick_id` is monotonic within run | Assigned by subquery `MAX(tick_id)+1` |

---

## Freshness Model (Framework §17.5)

The tick ledger is the freshness anchor for the HLE. A verifier can determine whether
evidence is "fresh" relative to a run by checking:

- Is there a `VerifierVerdictRecorded` tick with `created_unix` within the run window?
- Does the causal chain from the verdict tick trace back to the `RunStart` tick without gaps?

The TTL predicate uses `now_ms() - retention_ms` expressed as:
```
created_unix < (strftime('%s','now') - retention_secs)
```

Retention is always passed as a bound parameter — never a hardcoded integer constant.

## Design Notes

1. **`tick_id` is run-scoped, not global.** The `id` column is the globally unique row
   identifier; `tick_id` is monotonic only within a run. Verifiers reference `id` for
   causality pointers, not `tick_id`.

2. **Recursive CTE for causal chain.** SQLite supports recursive CTEs (3.8.3+). The
   `causal_chain` method uses a recursive CTE to walk parent pointers in a single query
   rather than N+1 selects.

3. **No time calls inside the store.** `created_unix` is always caller-supplied.

4. **STRICT table.** The `workflow_ticks` table uses SQLite's STRICT mode (available
   3.37.0+) to enforce column type affinity at the storage layer.

## Test Targets (minimum 50)

- `insert_run_start_no_parent`: RunStart tick with null parent inserts successfully
- `insert_step_start_with_parent`: StepStart with valid parent_tick_id succeeds
- `insert_parent_missing_rejected`: TickParentMissing (2431) for nonexistent parent
- `insert_parent_wrong_run_rejected`: parent from different run is rejected
- `list_for_run_ordered_by_tick_id`: returned ticks are ascending by tick_id
- `list_for_run_empty`: returns empty vec for run with no ticks
- `latest_for_run_returns_last`: latest_for_run returns tick with max tick_id
- `latest_for_run_empty`: returns Ok(None) for new run
- `causal_chain_single_tick`: chain of length 1 for RunStart
- `causal_chain_full_run`: reconstructs RunStart -> StepStart -> StepComplete
- `causal_chain_broken_link`: TickParentMissing if a link is missing
- `count_by_kind_step_complete`: counts only StepComplete ticks
- `count_by_kind_zero`: returns 0 for kind not present
- `tick_id_monotonic`: sequential inserts have strictly increasing tick_id
- `delete_older_than_removes_old`: old ticks deleted by TTL predicate
- `delete_older_than_spares_recent`: recent ticks not deleted
